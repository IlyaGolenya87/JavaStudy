http - hyper text ... protocol
Клавиша F12 в браузере - данные для разработчика
XHR - данные, передающиеся от сайта к серверу
Request metods:
- получить данныеы - GET
- отправить данные - POST
- удалить - DELETE 

Responce headers - ответы от сервера
Request headers - запросы на сервер

Протокол TCP - данные либо не дойдут либо дойдут полностью (проверяются хэш-суммы) - медленный, но надежный, используется
по умолчанию при стандартной конфигурации сети.
Протокол UDP - данные доходят в любом случае (без проверки хэш-суммы) - быстрый, но ненадежный

xml и html - форматы передачи данных между клиентом и сервером (информация разбиватеся по тегам <> и в каждом теге
помещаются необходимые данные)

В языках программирования содержатся библиотеки, предназначенные для расшифровки данных получаемых в различных форматах

формат json - новый формат, на смену xml 

РАБОТА С ФОРМАТОМ JSON:
-(JavaScript Object Notation)

Информация передается в виде объектов. 
{} - объявление объекта;
[] - объявление массива;

Типы данных:
- строка; "String" - объявление строкового значения
- число; 123 - объявление числового значения
- логическое занчение (boolean); true/false - объявление логического значения

ПРИМЕР:
{
	"name": "Vasya",  	//свойство пишется в кавычках, затем двоеточие и значение свойства, затем ставится запятая(если это не последнее свойство)
	"age" : 45,
	"isMarried" : true,
	"favoriteDrinks" : ["Cola", "Sprite", "Pepsi"]
	"passport" : {
		"number" : "KB2077647"
		"exireDate" : "20/11/2021"
	}


}
